Observability Mastery

Understand the technical intricacies behind the metrics. Pingtym Insights uses high-precision algorithms to decode how the web actually works.

D
DNS Resolution

Translates human-readable names into IP addresses. Pingtym tracks DNSStart and DNSDone events to isolate slow registrar lookups.

"Like looking up a name in a phonebook."
trace.DNSStart → trace.DNSDone
T
TLS Handshake

Negotiates secure encryption keys. We measure the time between the TCP connection and the final TLSHandshakeDone event.

"Spies exchanging secret code-books."
trace.TLSHandshakeStart → Done
B
TTFB (Emerald)

The time your server spends 'thinking' before sending the first byte. This is the ultimate measure of server-side efficiency.

"Time between ordering and the appetizer arriving."
request.Send → trace.GotFirstResponseByte
K
Keep-Alive Pooling

Instead of reconnecting, we reuse open sockets. This identifies the 'Warm Start' performance of your app during real user sessions.

"Staying on the line with a friend."
http.Transport.MaxIdleConns = 100
S
SaaS Correlation

We correlate your local Degraded status with upstream provider JSON APIs. This tells you if the internet is broken, or just your code.

GET /api/v2/summary.json
M
Monotonic Timing

Standard clocks can drift. Pingtym uses hardware-level monotonic timers to measure physical time elapsed with sub-millisecond accuracy.

time.Since() uses Monotonic Clock
